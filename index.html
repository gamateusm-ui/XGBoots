Algoritmos Avanzados

eXtreme Gradient Boosting (XGBoost)

Una biblioteca de potenciación de gradiente distribuida y optimizada, diseñada para ser altamente eficiente, flexible y portátil[cite: 2].

📌 ¿Qué es XGBoost?

XGBoost es la abreviatura de "extreme gradient boosting"[cite: 6]. Implementa algoritmos de aprendizaje automático bajo el marco de potenciación de gradiente [cite: 3] y es un método supervisado ideal para tareas de clasificación y regresión[cite: 5].

A diferencia de otros métodos como los bosques aleatorios, donde los árboles se entrenan de forma independiente [cite: 95], XGBoost funciona de manera secuencial: combina muchos aprendices débiles para formar un modelo fuerte [cite: 90], donde cada nuevo árbol corrige los errores o residuales de las iteraciones anteriores[cite: 91, 110].

⚡ Características de Alto Rendimiento

Computación Distribuida

Almacena datos en unidades de memoria llamadas bloques[cite: 11]. Permite el entrenamiento distribuido en clústeres usando herramientas como Apache Spark, Dask o Kubernetes[cite: 13, 14].

Precarga en Caché

Emplea un algoritmo de precarga que reduce drásticamente el tiempo de ejecución [cite: 15], permitiendo procesar miles de millones de ejemplos de forma escalable[cite: 17].

Regularización Integrada

Incluye regularización matemática como parte del objetivo de aprendizaje (L1 y L2) para evitar de forma directa el sobreajuste del modelo[cite: 18, 93, 118].

Manejo de Valores Faltantes

Utiliza un algoritmo consciente de la escasez para datos dispersos. Aprende de manera automática la mejor dirección predeterminada para clasificar los missing values[cite: 21, 22].

🎛️ Hiperparámetros Críticos

eta (Learning Rate):

Controla el tamaño de paso o contracción del algoritmo (por defecto es 0.3)[cite: 43, 46]. Un valor menor ayuda a evitar el sobreajuste al ralentizar el aprendizaje[cite: 48].

n_estimators:

Especifica el número de árboles secuenciales a construir[cite: 50]. Añadir demasiados puede generar sobreajuste, por lo que suele equilibrarse bajando la tasa eta[cite: 54].

max_depth:

Representa la profundidad máxima que puede alcanzar cada árbol (por defecto es 6)[cite: 60, 63]. Aumentar este valor incrementa la complejidad del modelo y el riesgo de sobreajuste[cite: 62].

gamma:

Controla la cantidad mínima de reducción de pérdida requerida para realizar una división adicional en un nodo[cite: 55]. Funciona como un parámetro de poda[cite: 117].
